Session Sharing

This service gives you the ability to share you session with other users. They will connect directly into the same remote desktop session you are connected to and will have joint control of the mouse and keyboard. WARNING: Any user that you share your machine with will have complete access and permissions to all of your files. You should supervise any shared access and remove users once sharing is no longer required.

SSH

To SSH into a machine you now need to use your user office credentials
Via terminal you can use ssh -l "forname.surname@email.com" host-140-256-215-209.nubes.stfc.ac.uk to avoid needing to escape symbols in your email.

Download/Upload Files

The easiest way to download and upload data is by using the new file browser located at data.analysis.stfc.ac.uk
It uses the same credentials as this page (your user office email and password). Once logged in you can access your processed data by clicking "Data" on the left menu and access raw data by clicking "Experiments".

You can download and upload your files via SFTP. The easiest way to do this is using an SFTP client like FileZilla
When connecting using an SFTP client:

  • The host name is the host listed next to the machine on your analysis dashboard.
  • Use the full host name (e.g. host-130-246-213-71.nubes.stfc.ac.uk) or else it will not work
  • Username and Password are your user office email address and password (the same as when you RDP into the machine)
  • The Port is 22 WARNING: If you do not specify the port, it will not work

Trouble Connecting?

If you're having trouble connecting, it may be because the network you're using to connect doesn't allow outbound connections on WebSockify or RDP Ports. If you're connecting using a proxy, you may have difficulties if it doesn't support websockets. You may need to contact your local network administrator or institution, to request they allow outbound traffic on port 3389 (RDP)

I'm having trouble deleting files.

If you're having trouble deleting files, you can hold shift then press delete, or rightclick > permanently delete to remove the files. WARNING: Files deleted this way will not be recoverable.

How long do the sessions last for?

The sessions currently expire after 72h of inactivity. If you need longer, please contact IDAaaS support.

What do I do if no machines are available?

Hopefully there should always be machines available, but if there aren't any, please wait 10-15 minutes and try again. If all the machines have been taken, it takes a few minutes to create some more. If you're still seeing that there are no machine available, please contact IDAaaS support.

How do I resize the session?

If you're using the browser, it will automatically resize to fit the window. If you're using RDP, you need to click Applications > System > Resolution. In the display window that opens you can adjust the resolution of the session.

How do I share my analysis environment?

Next to your analysis environment's "Launch" button is the "Share" button. Click share, search for the user you want to share with, click their name, then click "Save Changes".

Which credentials do I use to log in via RDP?

For logging in via RDP, you need to use your user office email and password. See here for what you should do if you've forgotten it.

How do I reset my Password?

To reset your password please go to User Office.

How do I get a user office account?

You can register for a user office account here.

How do I copy & paste?

Currently copy/paste to and from the analysis environments only works when using the browser (There is work planned to implement copy/paste on RDP too).

You can copy and paste using the shortcuts you're used to on your computer, CTRL + C, CTRL + V, CTRL + SHIFT + V, SHIFT + INS all work on Chrome and Safari.

Firefox doesn't allow SHIFT + INS, so if you want to paste into the terminal on your analysis environment, you will need to use the top bar.

Top Bar

If you move your cursor towards the middle of the top of your window, a bar will drop down.


Button 1 opens the copy menu, Button 2 toggles between fullscreen and windowed, and Button 3 opens the paste menu



Any text that you highlight in your analysis environment will be added to the copy bar so that you can copy it.



To use the paste menu, click where you want the text to go in your analysis environment, then paste the your text into the paste menu, then press the paste button.

How and where is my data stored?

The ISIS Archive

(same as \\isis.cclrc.ac.uk\inst$\) Contains all historic experiment data and is mounted read-only, so nobody will be able to write anything here. There are shortcuts on the desktop and in the applications menu, and you can go to /archive/.

Ceph

This holds all the home directories, RB directories, and auxiliary directories.
  • Home directories are your own personal space that other users don't have access to. They can be found at /home/

  • RB directories are automatically created for every experiment/investigation with an RB, they are divided by instrument and can be found in /instrument/INSTRUMENT-NAME/RBNumber/. They can also be seen on a per cycle basis in /instrument/INSTRUMENT-NAME/CYCLE20###/. Instrument scientists have read-write access to all the RB directories for their instruments. Regular users will have access to the RB directory corresponding to their experiment provided they are listed on the proposal. We can add people to experiments later to give them permission to access the directories, just send us an email with the name and email address of the user. There are shortcuts to these RB directories on the desktop and in the applications menu. Any files which are related to the experiment should be in here, so you can work on them together.

  • Auxiliary directories, are located here /mnt/auxiliary/ceph/<instrument group> Instrument Scientists have read write access to this folder, regular users have read only access. The purpose of this directory is to give Instrument scientists a place to store any common files they would like to distribute to the users.

Babylon

Babylon is a temporary network storage space run by ISIS located at /babylon/

Persistence

Anything saved on Ceph (home dirs, RB dirs, auxiliary dirs) is safe and won't be lost if you delete your session. Anything stored outside these directories, for example in /tmp/ or /opt/ will only exist on that session and cannot be recovered if the session is deleted.
